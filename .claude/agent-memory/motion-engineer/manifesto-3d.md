# Manifesto WebGL MacBook scroll-story + R3F↔GSAP pattern (as-built 2026-07-13, PROMPT #4)

Supersedes the keyboard disintegration (`376f4c0`) and pinned scroll-fill (`c14474e`) — both in git history.

## Scroll architecture (ManifestoSection.tsx — ONE useGSAP, deps [prefersReducedMotion], revertOnUpdate)

- **T1 seam** `{trigger:"#hero", start:"clamp(bottom bottom)", end:"+=120%", pin, scrub, invalidateOnRefresh}`: the fixed stage's `clip-path` expands from the hero slot rect to `inset(0px 0px 0px 0px round 0px)` — from/to MUST be shape-matched (same token count/units) or the interpolation snaps. Name rows `y` off ±(slot edge + row height) by 0.9; `.hero-item, .hero-bar` fade by 0.35 with `immediateRender:false` (tolerates scrolling mid-hero-intro); glow 0.35→1; `channels.sceneIntro` 0→1; `.hero-name` x/y→0 over the first 5% (kills parallax residue).
- **T2 master** `{trigger:"#manifesto" (520vh, -mt-[100svh]), start:"top top", end:"bottom bottom", scrub}` — **NO invalidateOnRefresh**: plain-object channel tweens re-record drifted starts mid-scrub on resize. All `fromTo`s with explicit values at phase fractions: P1 hold 0–.08 · lid .08–.34 · logo up .10/down .34 · spill .20 · yaw .34–.60 · screen full .42–.58 · recede+copy .58 · words .60–.82 (`stagger:{amount}`) · veil .84–1 (veil opacity + `--veil-blur` 0→24px + stage scale→1.05). Sequential same-property `fromTo` pairs need `immediateRender:false` on the second.
- **-mt-[100svh]** on `#manifesto` bridges the structural 100vh dead zone between T1's pin-spacer end and T2's `top top` (exact for any hero height; motion branch only — reduced motion has no spacer to compensate).
- **Seam measurement**: accumulate `offsetTop/offsetLeft` up the offsetParent chain to `#hero` + `min(0, innerHeight − hero.offsetHeight)` pin shift. Never gBCR (corrupts at the pin park position on mid-page refresh, inherits the ±10px parallax). The parallax transform makes `.hero-name` itself an offsetParent — a single hop measures against the wrong box. `ScrollTrigger.config({ignoreMobileResize:true})` + `document.fonts.ready→refresh()` live in `lib/gsap.ts`.
- **Stage lifecycle**: fixed z-20 stage hides ONLY past manifesto end via `autoAlpha` (never `display:none` — zero-sizes the GL buffer), derived from `progress>=1 && !isActive` on `onUpdate`+`onRefresh` (scrollRestoration-safe); `stageState.active` gates the render tick.
- **Backdrop-root trap**: the veil's `backdrop-filter: blur(var(--veil-blur))` stops sampling the canvas if ANY ancestor gets `opacity<1` — animate only the veil's own opacity. The sticky inner needs explicit `z-30` (pinned/sticky wrappers are z-auto stacking contexts that otherwise paint under the z-20 fixed stage).
- Statement = pre-split words in JSX (module-level split of `profile.manifesto.lines`; focal-word test strips punctuation). Hero side: SplitType splits the two row spans individually (never the h1 — revert would destroy the slot node); parallax handler early-returns past `scrollY > 8`.
- **Veil perf**: measured 121 fps (P2–P3) / 105 fps (veil) at dpr 2 — animated backdrop-filter kept; the freeze-canvas + `filter:blur()` fallback exists as a documented option but was not needed.

## R3F island (src/features/home/components/manifesto-3d/ — reuse for any future 3D)

- Stack three/@react-three/fiber@9/@react-three/drei@10 (+@types/three); React 19 needs fiber v9. `ManifestoCanvas.tsx` is the lazy boundary — the section may import ONLY `channels.ts` (dep-free singleton) + `ModelErrorBoundary`, or the ~960 kB three chunk leaks into main. `import { advance }` in a section file defeats the split too.
- **RULE 3**: `<Canvas frameloop="never">` + `FrameDriver` inside the chunk: `gsap.ticker.add(t => advance(t*1000))` (per-root `useThree(s=>s.advance)`; **ms** timestamps — the Lenis `raf(t*1000)` convention); tick early-returns on `document.hidden` / `!stageState.active`.
- **Choreography as data**: GSAP tweens `channels` only; `MacbookModel`'s `useFrame` damps rendered values toward channels — `THREE.MathUtils.damp(x, target, 6, Math.min(delta, 1/30))`. useFrame delta is SECONDS (gsap.ticker deltaTime is MS and silently turns damp into a snap); the clamp prevents teleports after paused stretches. Timelines NEVER wait for the model: a late GLB damp-catches-up; GLB/WebGL/chunk failure keeps the whole DOM sequence (`ModelErrorBoundary` → P3 poster `src/assets/images/macbook-poster.webp`).
- Procedural `<Environment resolution={256} frames={1}>` + `<Lightformer>`s = ZERO network (the old "no drei Environment" rule targeted HDRI presets/files only). Ember directional reads `--color-accent` at mount (aurora precedent). Lighting is tuned dark — a big soft key ≥2 blows the silver shell white at mid-lid angles; keep key ~0.85 + thin top strip + cool fill 0.3.
- **Hinge rig** (`rig.ts`, Case A: named subtrees `BoBvWqDHZjAeVrp_44` base / `VCQqxpxkUlzqcJI_62` lid): pivot at the deck-line hinge via `parent.worldToLocal` + `attach()` (idempotency guard — StrictMode double-runs useMemo); open angle ≈1.955 rad measured from bboxes. **Sign is EMPIRICAL**: this asset closes at `+openAngle` (`-` folds the lid under the base) — baked chain matrices flip naive world-space math; lock as a constant after one screenshot. `CLOSED_TUCK 1.015` seats the lid flush (bbox hinge sits ~0.7 raw units front of the true barrel). **Never judge poses via `Box3.setFromObject` on a rotated subtree** — the loose raw AABB re-AABB'd inflates massively (cost two false "broken hinge" reads); use renders or world quaternions. Screen material `sfCQkHOWyrsLmor` ships `emissiveTexture` + KHR strength 8 → just drive `emissiveIntensity` (`toneMapped:false`); logo `CdgEAaPUlrQWQuD` gets white emissive ramped by `logoGlow`. Portrait fit clamp `min(1, viewport.width*0.92/FIT_SIZE)` (desktop FIT_SIZE 3.2 overflows a 390px frustum at ~183%).

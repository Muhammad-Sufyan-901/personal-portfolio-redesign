## R3F island (src/features/home/components/manifesto-3d/ — reuse for any future 3D)

- Stack three/@react-three/fiber@9/@react-three/drei@10 (+@types/three); React 19 needs fiber v9. `ManifestoCanvas.tsx` is the lazy boundary — the section may import ONLY `channels.ts` (dep-free singleton) + `ModelErrorBoundary`, or the ~960 kB three chunk leaks into main. `import { advance }` in a section file defeats the split too.
- **Transform-scaled Canvas ancestor ⇒ `resize={{ ..., offsetSize: true }}` is MANDATORY**: R3F sizes via react-use-measure, whose default reports getBoundingClientRect — transforms included. With the seam's `scale(0.14)` stage, the default sized the canvas to ~14% of the viewport anchored top-left (scene in the corner), only re-measuring up as scroll events chased the growing scale ("corner for a second, then snaps to center"). `offsetSize` measures layout dims (offsetWidth/Height), transform-immune — buffer stays full-res through the whole zoom and the veil's 1.05 scale.
- **RULE 3**: `<Canvas frameloop="never">` + `FrameDriver` inside the chunk: `gsap.ticker.add(t => advance(t))` (per-root `useThree(s=>s.advance)`). **fiber v9 `advance(timestamp)` takes SECONDS** — it feeds `state.clock` directly. Passing ms (`t*1000`, the Lenis `raf` convention — a wrong first guess that shipped) made `clock.elapsedTime` ms-as-seconds (~70,000) and every `elapsedTime`-driven sinusoid thrashed at ~2,750 rad/s = a violent nonstop shake; the damp path survived only because of the delta clamp. gsap ticker time (seconds) passes through unchanged; only gsap's ticker callback `deltaTime` arg is ms. Tick early-returns on `document.hidden` / `!stageState.active`.
- **Choreography as data**: GSAP tweens `channels` only; `MacbookModel`'s `useFrame` damps rendered values toward channels — `THREE.MathUtils.damp(x, target, DAMP_LAMBDA 4.5, Math.min(delta, 1/30))` (6→4.5 in the 2026-07-17 smoothness retune — the single feel knob for model + camera glide); the clamp prevents teleports after paused stretches. **First-frame correctness by construction** (late chunk/GLB, deep links, upward entry): `rendered` initializes FROM the live channels (never zeros — else the model damp-animates from the P1 pose to wherever the scroll is), the camera is aimed via `aimCamera()` in Canvas `onCreated` + at CameraRig mount (a default-orientation camera must never produce a frame), and the portrait fit clamp derives from `useThree(s=>s.size)` + CAM constants, not `state.viewport` (recompute-timing dependent). The object is PERFECTLY STILL between beats (idle sway/float removed by user request — every visible move is a damped channel transition). `CameraRig` damps its own `lidProgress` copy and tracks the machine's visual center (`CAM.centerClosed 0.1 → centerOpen 1.05`, `rise 0.6` = product pitch) so the MacBook stays screen-centered through the lid arc. Timelines NEVER wait for the model: a late GLB damp-catches-up; GLB/WebGL/chunk failure keeps the whole DOM sequence (`ModelErrorBoundary` → P3 poster `src/assets/images/macbook-poster.webp`).
- Procedural `<Environment resolution={256} frames={1}>` + `<Lightformer>`s = ZERO network (the old "no drei Environment" rule targeted HDRI presets/files only). Ember directional reads `--color-accent` at mount (aurora precedent). Lighting is tuned dark — a big soft key ≥2 blows the silver shell white at mid-lid angles; keep key ~0.85 + thin top strip + cool fill 0.3.

## Scroll architecture (ManifestoSection.tsx — ONE useGSAP, deps [prefersReducedMotion], revertOnUpdate)

- **T1 seam** `{trigger:"#hero", start:"clamp(bottom bottom)", end:"+=120%", pin, scrub, invalidateOnRefresh}`: stage starts `autoAlpha 0, scale SEAM_BOX_SCALE 0.14, transformOrigin 50% 50%` (CSS `opacity-0` pre-JS default) — nothing shows for the first `SEAM_APPEAR 0.42` of the pin (≈ 0.5·viewport of scroll, "halfway down the screen"), then: autoAlpha in over 0.08, scale 0.14→1 across the remainder, borderRadius (pre-divided by the scale so the small box's VISUAL rounding equals `--radius-lg`) → 0. Name rows exit 0.5–0.95 (distances via accumulated row offsets); `.hero-item, .hero-bar` fade 0.4–0.65 with `immediateRender:false`; glow 0.35→1; `channels.sceneIntro` from SEAM_APPEAR; `.hero-name` x/y→0 just before (kills parallax residue). NOTE: `fromTo`s positioned >0 inside a timeline auto-default `immediateRender:false`, so the later stage tweens don't stomp the initial small-box set.
- **T2 master** `{trigger:"#manifesto" (520vh, -mt-[100svh]), start:"top top", end:"bottom bottom", scrub}` — **NO invalidateOnRefresh**: plain-object channel tweens re-record drifted starts mid-scrub on resize. All `fromTo`s with explicit values at phase fractions: P1 hold 0–.08 · lid .08–.34 · logo up .10/down .34 · spill .20 · yaw .34–.60 · screen full .42–.58 · recede+copy .58 · words .60–.82 (`stagger:{amount}`) · veil .84–1 (veil opacity + `--veil-blur` 0→24px + stage scale→1.05). Sequential same-property `fromTo` pairs need `immediateRender:false` on the second.
- **-mt-[100svh]** on `#manifesto` bridges the structural 100vh dead zone between T1's pin-spacer end and T2's `top top` (exact for any hero height; motion branch only — reduced motion has no spacer to compensate).
- **Seam measurement** (row exit distances): accumulate `offsetTop` up the offsetParent chain to `#hero` + `min(0, innerHeight − hero.offsetHeight)` pin shift. Never gBCR (corrupts at the pin park position on mid-page refresh, inherits the ±10px parallax). The parallax transform makes `.hero-name` itself an offsetParent — a single hop measures against the wrong box. `ScrollTrigger.config({ignoreMobileResize:true})` + `document.fonts.ready→refresh()` live in `lib/gsap.ts`.
- **Stage lifecycle**: `stageState.active = seamStarted && !pastEnd` — the render loop runs ONLY while the stage could be seen (pre-warmed 0.1 seam-progress before the box appears; nothing renders behind the hero at rest). Alpha: T1's scrub owns the appearance; the T2 handler touches it only when `!visible || progress > 0` (else a T2 flip at rest could force the hidden stage visible for a frame). Hide via `autoAlpha`, never `display:none` (zero-sizes the GL buffer); derive from `progress>=1 && !isActive` on `onUpdate`+`onRefresh` (scrollRestoration-safe). Declare the lifecycle closures BEFORE both triggers — their `onRefresh` can fire during creation (TDZ).
- **Backdrop-root trap**: the veil's `backdrop-filter: blur(var(--veil-blur))` stops sampling the canvas if ANY ancestor gets `opacity<1` — animate only the veil's own opacity. The sticky inner needs explicit `z-30` (pinned/sticky wrappers are z-auto stacking contexts that otherwise paint under the z-20 fixed stage).
- Statement = pre-split words in JSX (module-level split of `profile.manifesto.lines`; focal-word test strips punctuation). Hero side: SplitType splits the two row spans individually (keeps the h1 unrebuild by reverts; the seam animates the row spans); parallax handler early-returns past `scrollY > 8`.
- **Veil perf**: measured 121 fps (P2–P3) / 105 fps (veil) at dpr 2 — animated backdrop-filter kept; the freeze-canvas + `filter:blur()` fallback exists as a documented option but was not needed.

## R3F island (src/features/home/components/manifesto-3d/ — reuse for any future 3D)

- Stack three/@react-three/fiber@9/@react-three/drei@10 (+@types/three); React 19 needs fiber v9. `ManifestoCanvas.tsx` is the lazy boundary — the section may import ONLY `channels.ts` (dep-free singleton) + `ModelErrorBoundary`, or the ~960 kB three chunk leaks into main. `import { advance }` in a section file defeats the split too.
- **Transform-scaled Canvas ancestor ⇒ `resize={{ ..., offsetSize: true }}` is MANDATORY**: R3F sizes via react-use-measure, whose default reports getBoundingClientRect — transforms included. With the seam's `scale(0.14)` stage, the default sized the canvas to ~14% of the viewport anchored top-left (scene in the corner), only re-measuring up as scroll events chased the growing scale ("corner for a second, then snaps to center"). `offsetSize` measures layout dims (offsetWidth/Height), transform-immune — buffer stays full-res through the whole zoom and the veil's 1.05 scale.
- **RULE 3**: `<Canvas frameloop="never">` + `FrameDriver` inside the chunk: `gsap.ticker.add(t => advance(t))` (per-root `useThree(s=>s.advance)`). **fiber v9 `advance(timestamp)` takes SECONDS** — it feeds `state.clock` directly. Passing ms (`t*1000`, the Lenis `raf` convention — a wrong first guess that shipped) made `clock.elapsedTime` ms-as-seconds (~70,000) and every `elapsedTime`-driven sinusoid thrashed at ~2,750 rad/s = a violent nonstop shake; the damp path survived only because of the delta clamp. gsap ticker time (seconds) passes through unchanged; only gsap's ticker callback `deltaTime` arg is ms. Tick early-returns on `document.hidden` / `!stageState.active`.
- **Choreography as data**: GSAP tweens `channels` only; `MacbookModel`'s `useFrame` damps rendered values toward channels — `THREE.MathUtils.damp(x, target, 6, Math.min(delta, 1/30))`; the clamp prevents teleports after paused stretches. **First-frame correctness by construction** (late chunk/GLB, deep links, upward entry): `rendered` initializes FROM the live channels (never zeros — else the model damp-animates from the P1 pose to wherever the scroll is), the camera is aimed via `aimCamera()` in Canvas `onCreated` + at CameraRig mount (a default-orientation camera must never produce a frame), and the portrait fit clamp derives from `useThree(s=>s.size)` + CAM constants, not `state.viewport` (recompute-timing dependent). The object is PERFECTLY STILL between beats (idle sway/float removed by user request — every visible move is a damped channel transition). `CameraRig` damps its own `lidProgress` copy and tracks the machine's visual center (`CAM.centerClosed 0.1 → centerOpen 1.05`, `rise 0.6` = product pitch) so the MacBook stays screen-centered through the lid arc. Timelines NEVER wait for the model: a late GLB damp-catches-up; GLB/WebGL/chunk failure keeps the whole DOM sequence (`ModelErrorBoundary` → P3 poster `src/assets/images/macbook-poster.webp`).
- Procedural `<Environment resolution={256} frames={1}>` + `<Lightformer>`s = ZERO network (the old "no drei Environment" rule targeted HDRI presets/files only). Ember directional reads `--color-accent` at mount (aurora precedent). Lighting is tuned dark — a big soft key ≥2 blows the silver shell white at mid-lid angles; keep key ~0.85 + thin top strip + cool fill 0.3.
- **Hinge rig** (`rig.ts`, Case A: named subtrees `BoBvWqDHZjAeVrp_44` base / `VCQqxpxkUlzqcJI_62` lid): pivot at the deck-line hinge via `parent.worldToLocal` + `attach()` (idempotency guard — StrictMode double-runs useMemo); open angle ≈1.955 rad measured from bboxes. **Sign is EMPIRICAL**: this asset closes at `+openAngle` (`-` folds the lid under the base) — baked chain matrices flip naive world-space math; lock as a constant after one screenshot. `CLOSED_TUCK 1.015` seats the lid flush (bbox hinge sits ~0.7 raw units front of the true barrel). **Never judge poses via `Box3.setFromObject` on a rotated subtree** — the loose raw AABB re-AABB'd inflates massively (cost two false "broken hinge" reads); use renders or world quaternions. Screen material `sfCQkHOWyrsLmor` ships `emissiveTexture` + KHR strength 8 → just drive `emissiveIntensity` (`toneMapped:false`); logo `CdgEAaPUlrQWQuD` gets white emissive ramped by `logoGlow`. Portrait fit clamp `min(1, viewport.width*0.92/FIT_SIZE)` (desktop FIT_SIZE 3.2 overflows a 390px frustum at ~183%).

## Notable decisions

- **Rows not wrapped in `RevealText`:** split-type does not support nested elements; each row is a flex of name + level spans, so SplitType line-grouping would reparent the words and destroy the layout. Instead `PillarBlock` replays the exact RevealText recipe (`yPercent: 100`, `overflow-hidden` clip on the `li`, stagger 0.08, `once` trigger, reduced-motion skip) in a scoped `useGSAP` — same precedent as `ManifestoSection`.
- **Levels cross-referenced, never invented:** pillar entries are plain strings; levels come from PRD `skills` by exact name plus a 2-entry alias map (`React → React JS`, `Tailwind → Tailwind CSS`). `shadcn/ui` and `Livewire` have no skillList entry → no level shown.
- **Full-bleed marquee via `-mx-page-x`:** Tailwind v4 generates negative margin utilities from the `--spacing-page-x` token — verified in dist CSS. Pillar gutter reuses the same token (`gap-x-page-x`) instead of a bespoke clamp.
- Marquee content is one span of `skills.map(name).join(" · ")` with a trailing separator so the loop seam reads continuously; duplicate track is `aria-hidden` by the primitive.

## Notable decisions

- **Scroll-activation over hover** (owner decision b) — one mechanism for all
  inputs; deletes the `(hover:hover)` split and answers mobile (preview column
  hidden < lg, like the reference's mobile).
- **Pixel-space path `d`** — the section measures the path layer
  (ResizeObserver) and scales the tunables' normalized `d` into a 1:1 px
  viewBox (`scalePathD`). Chrome renders screen-space dashes under
  `non-scaling-stroke`, which tiles a stretched viewBox into disconnected
  capsule segments (QA blocker #1).
- **`pathLength={1000}`, not `1`** — GSAP's CSS layer rounds
  `stroke-dashoffset` to whole px, so a 0–1 range quantizes into a binary
  step at 50% scrub (found via live trigger inspection; draw now interpolates
  at 0.1% granularity).
- **No `revertOnUpdate` on the crossfade effect** (QA #2) — reverting snapped
  the outgoing layer to CSS-hidden and flashed the preview through black;
  `overwrite: "auto"` + `staticMode`'s `hidden` class cover cleanup.
- `transition-[color,scale]` not `…,transform]` — Tailwind v4 scale utilities
  set the standalone `scale` property (QA #3).
- QA nit #6 (description expand shifts lower rows' cached focal bands by one
  description height) accepted — self-correcting at each handoff.
